Connect to Your Server (a.k.a. "Remote Control Time")

SSH is like a text-based remote control for your server. Don't worry, it's less scary than it sounds.

🎮 SSH Explained Like You're Five

Imagine your server is a TV in another building. SSH is the universal remote that lets you change channels from your house. Except instead of changing channels, you're typing commands, and instead of a TV, it's a computer. Okay, the analogy broke down, but you get the idea. Now you know.

For Mac/Linux Users:

Open Terminal (it's in Applications → Utilities, or press Cmd+Space and type "Terminal")

First, make your SSH key secure (SSH is paranoid about file permissions):

chmod 400 /path/to/your-downloaded-key.key

Then connect:

ssh -i /path/to/your-downloaded-key.key ubuntu@YOUR_SERVER_IP

For Windows Users:

Open PowerShell (right-click Start button → Windows PowerShell)

Use the same commands as above! Windows finally caught up.

✅ Success Check: If you see something like ubuntu@your-server-name:~$, you're in! You did it! Take a moment to feel like a hacker in a movie.

Prepare Your Server (a.k.a. "Installing the Basics")

Fresh servers are like new phones - they need updates before they're useful.

Update Everything

sudo apt update && sudo apt upgrade -y

This says: "Hey Ubuntu, check for updates and install them. Say yes to everything." It'll take a few minutes - perfect time for a coffee break!

Install Essential Tools

sudo apt install -y git curl python3 python3-pip python3-venv
Tool What It Does
git Downloads code from the internet (like an app store for nerds)
curl Downloads files from web links
python3 The programming language that runs everything
python3-pip Python's package manager (app store for Python)
python3-venv Keeps Python projects separate so they don't fight
💡 Pro Tip from @aiwithnaman: These tools are the foundation of almost every cloud project. Learn what each does and you'll be ahead of 90% of beginners. Now you know.

Turn On the Firewall (Lock the Door)

⚠️ CRITICAL: Run these commands in EXACTLY this order! If you enable the firewall before allowing SSH, you'll lock yourself out. Then you'll have to rebuild your server and explain to your friends why you disappeared for an hour.
sudo ufw allow ssh sudo ufw enable

Check if it's working:

sudo ufw status

Install Docker (a.k.a. "The Apartment Manager")

🏢 What is Docker?

Docker creates isolated "containers" - think of them as apartments in a building. Each apartment has its own kitchen, furniture, and rules. What happens in Apartment A doesn't affect Apartment B. Your server is the building, and Docker is the super. Now you know.

Install Docker

sudo apt install -y docker.io sudo systemctl start docker sudo systemctl enable docker

Make Docker Play Nice (No More sudo Every Time)

sudo usermod -aG docker $USER exit

Then reconnect with SSH (same command as before). After that, you can run docker ps without sudo!

docker ps
✅ If you see a table (even if empty), Docker is working! If you see "permission denied", you forgot to log out and back in. We all have those days.

Get Your Free Gemini API Key (a.k.a. "The Brain Password")

An API key is like a VIP pass that lets your server talk to Google's AI. Without it, your server is just a very expensive calculator.

Getting the Key

  1. Go to aistudio.google.com
  2. Sign in with your Google account
  3. Click "Get API key" in the left sidebar

  5. Copy the key that appears - it looks like random gibberish (that's good!)
🔒 Guard This With Your Life! Your API key is like a credit card number. Don't paste it in emails, chats, or public forums. If someone gets it, they can use your AI quota. Not cool.

Install OpenClaw (a.k.a. "The Ringmaster")

🎪 What is OpenClaw?

OpenClaw is the manager that sits between you, Telegram, Gemini, and your agents. It's like a really efficient secretary who takes your messages, asks the AI for help when needed, and delegates tasks to the right "agent" (container).

We install it directly on the server (not in a container) so it can see and control all the containers. It's the boss with keys to every office. Now you know.

Install It

curl -fsSL https://openclaw.ai/install.sh | bash

Reload your terminal:

source ~/.bashrc

Check if it worked:

openclaw --version

Set Up Your AI Model

See what models are available:

openclaw models

Set Gemini as your model (copy the EXACT name from the list):

openclaw models set google/gemini-2.5-flash-lite
⚠️ Exact Names Only! OpenClaw is picky about model names. If you type "gemini-2.5-flash-lite-preview" when it wants "gemini-2.5-flash-lite", it will throw a tantrum. Copy-paste is your friend.

Add Your API Key

openclaw config set gemini_api_key YOUR_API_KEY_HERE

Replace YOUR_API_KEY_HERE with the key you got in Step 5. Obviously.

7

Create Your Agent Containers (a.k.a. "Building the Offices")

We're making two agents: agent-alpha and agent-beta. Why two? So you can see how routing works! You could make one, or ten - the process is the same.

Create Workspace Folders

mkdir -p ~/server/agent-alpha/tools mkdir -p ~/server/agent-beta/tools

Create Agent Alpha (The First Apartment)

docker run -d \ --name agent-alpha \ --restart unless-stopped \ -p 18790:18790 \ -v ~/server/agent-alpha:/workspace \ ubuntu:22.04 \ sleep infinity

Create Agent Beta (The Second Apartment)

docker run -d \ --name agent-beta \ --restart unless-stopped \ -p 18791:18791 \ -v ~/server/agent-beta:/workspace \ ubuntu:22.04 \ sleep infinity
📋 What This Does: Creates isolated containers that restart automatically if they crash, maps ports so OpenClaw can talk to them, and connects folders so files persist even if the container restarts.

Check if they're running:

docker ps

Install Tools Inside Each Container

Enter agent-alpha and install tools:

docker exec -it agent-alpha bash apt update && apt install -y curl python3 python3-pip exit

Do the same for agent-beta:

docker exec -it agent-beta bash apt update && apt install -y curl python3 python3-pip exit

Register Agents with OpenClaw (a.k.a. "Hiring the Staff")

The containers exist, but OpenClaw doesn't know about them yet. Let's introduce them!

Register Agent Alpha

openclaw agents register \ --name agent-alpha \ --type docker \ --container agent-alpha \ --workdir /workspace \ --port 18790

Register Agent Beta

openclaw agents register \ --name agent-beta \ --type docker \ --container agent-beta \ --workdir /workspace \ --port 18791

Verify Registration

openclaw agents list
✅ Both should show as "available". If they do, congratulations! You've got two obedient digital servants ready to do your bidding.

Set Up Telegram (a.k.a. "Your Remote Control")

BotFather is Telegram's official bot for creating bots. Yes, it's bots all the way down.

Create Your Bot

  1. Open Telegram and search for @BotFather
  2. Start a chat and send: /newbot
  3. Name it something like: My AI Server
  4. Username must end in 'bot', like: myaiserver_bot
  5. BotFather will give you a token (long string of random characters). Save it!
🔒 Keep This Token Secret! Anyone with this token controls your bot. Don't share it, don't commit it to GitHub, don't tattoo it on your arm (okay, maybe that last one is fine, but cover it up).

Connect Telegram to OpenClaw

Start the OpenClaw gateway temporarily:

openclaw gateway --port 18789

In a new terminal window (new SSH session), add your bot:

openclaw channels add telegram --token YOUR_BOT_TOKEN

Now open Telegram, find your bot, and send it any message (like "hello"). A pairing code will appear in your server terminal. Approve it:

openclaw pairing approve telegram PAIRING_CODE
🎉 You Did It! Your Telegram bot is now connected. You can send messages to your server from anywhere in the world! Now you know.

Make It Run Forever (a.k.a. "The Immortality Step")

Right now, if you close your laptop, everything stops. Let's fix that with systemd - Ubuntu's built-in nanny that keeps things running.

👶 What is systemd?

Think of systemd as a digital nanny. You tell it "watch this process and make sure it stays running", and it does exactly that. Server reboots? systemd restarts your stuff. Crashes? systemd restarts it. It's the helicopter parent of process managers. Now you know.

Create the Service File

First, stop any running gateway (Ctrl+C if it's running), then:

sudo nano /etc/systemd/system/openclaw.service

Paste this exactly:

[Unit] Description=OpenClaw AI Gateway After=network.target docker.service Requires=docker.service [Service] Type=simple User=ubuntu WorkingDirectory=/home/ubuntu ExecStart=/usr/local/bin/openclaw gateway --port 18789 Restart=always RestartSec=10 [Install] WantedBy=multi-user.target

Save: Ctrl+X, then Y, then Enter.

Enable and Start

sudo systemctl daemon-reload sudo systemctl enable openclaw sudo systemctl start openclaw

Check Everything

sudo systemctl status openclaw docker ps
✅ OpenClaw should show "active (running)" in green, and both containers should show "Up". Your system is now immortal! (Well, until the heat death of the universe, or Oracle changes their free tier policy, whichever comes first.)

When Things Go Wrong (a.k.a. "The Panic Room")

🔥 "Out of Capacity" Error

ARM servers are popular. Try a different availability domain, or wait a few hours. Persistence!

🔥 "Permission denied (publickey)"

chmod 400 /path/to/your-key.key

🔥 "Docker permission denied"

sudo usermod -aG docker $USER # Then log out and back in

🔥 Locked yourself out with UFW

Use Oracle's browser-based console (Cloud Shell) to access your server and fix the firewall. We've all been there.

🔥 Container shows "Exited"

docker logs agent-alpha --tail 30 docker start agent-alpha
